# 轉接人工規則｜escalation-rules.md

## 核心原則

轉接不是失敗，是保護顧客體驗的最後一道防線。
寧可多轉一個，也不要讓顧客在 AI 面前空轉。

---

## 七個觸發條件

### 觸發 1：關鍵字偵測（即時觸發，不經 LLM）

以下關鍵字出現在顧客訊息中，直接觸發轉接，不需要 LLM 判斷：

```python
ESCALATION_KEYWORDS = [
    # 要求找真人
    "找主管", "找你們主管", "找負責人", "找店長",
    "真人客服", "人工客服", "找真人", "不要跟機器人講",
    "轉接", "轉真人",
    
    # 法律威脅
    "消保官", "消基會", "投訴", "要投訴", "申訴",
    "律師", "告你們", "法院", "公平會",
    
    # 強烈負面情緒
    "詐騙", "騙子", "爛透了", "什麼態度",
    "太誇張", "太扯了", "受不了",
]
```

### 觸發 2：連續不滿意（計數器）

顧客連續 2 次表達不滿意 AI 的回覆時觸發。

判斷「不滿意」的訊號：
- 「不是我問的」「你沒回答到我的問題」
- 「我剛才就說過了」
- 「你到底有沒有在聽」
- 重複問同一個問題（表示上一次回覆沒解決）

計數邏輯：
- 每次偵測到不滿意 → 計數 +1
- 顧客表達滿意或換話題 → 計數歸零
- 計數 ≥ 2 → 觸發轉接

### 觸發 3：退款金額爭議

當對話中同時出現以下條件：
- 提到「退款」「退費」「退錢」
- 提到具體金額或「金額不對」「少退」「多扣」

退款金額涉及財務操作，AI 沒有權限處理，必須轉人工。

### 觸發 4：需要寫入操作

以下需求超出 AI 的唯讀權限：
- 修改訂單（改地址、改商品、改尺寸）
- 取消訂單
- 執行退款
- 修改會員資料
- 訂製尺寸（需要人工確認規格和報價）

### 觸發 5：AI 信心不足

當 LLM 在回覆中使用以下表述時，Brain 應判斷為信心不足：
- 「我不太確定」
- 「建議你確認一下」
- 「這個部分可能需要」

如果連續 2 次出現信心不足的回覆 → 轉接。

### 觸發 6：敏感話題

- 顧客提到身體受傷（「睡你們的床受傷了」「腰更痛了」）
- 涉及兒童安全（「小朋友從床上摔下來」）
- 涉及法律責任的問題

### 觸發 7：顧客主動要求

顧客說「我要找真人」「讓真人跟我說」→ 無條件轉接。
不要試圖挽留，不要說「我也可以幫你」。

## 轉接話術（依場景分類）

### 一般轉接（觸發 2, 4, 5）
```
了解了，這個部分我想請更專業的輔睡員來幫你處理，會比較完整。
我幫你轉接，請稍等一下喔！
```

### 情緒轉接（觸發 1, 3, 6）
```
我理解你的感受，真的很抱歉讓你有這樣的體驗。
我馬上請輔睡員來幫你處理，他們會盡快跟你聯繫。
```

### 主動要求轉接（觸發 7）
```
好的，沒問題！我幫你轉接真人輔睡員，請稍等一下～
```

### 退換貨完成收集後的轉接
```
好的，資訊我都記下來了。我幫你提交退換貨申請，
輔睡員會在 1~2 個工作天內跟你聯繫安排後續喔！
有什麼問題隨時跟我說。
```

## 轉接時傳遞給人工的資訊包

AI 觸發轉接時，必須打包以下資訊傳給 Omnichat / 內部通知：

```json
{
  "line_uid": "U1234567890",
  "member_name": "王小明",
  "member_id": "cust_abc123",
  "escalation_trigger": "關鍵字：找主管",
  "intent": "COMPLAINT",
  "conversation_summary": "顧客詢問訂單 #LF2026001 的配送進度，表示等了三週還沒收到。情緒逐漸升高，要求找主管處理。",
  "last_5_messages": [
    {"role": "user", "content": "..."},
    {"role": "assistant", "content": "..."},
    ...
  ],
  "order_info": "訂單 #LF2026001，山丘樂眠床 5 尺，3/15 下單，狀態：製作中",
  "timestamp": "2026-04-07T14:30:00+08:00"
}
```

重點：讓接手的真人輔睡員不需要重新問一遍，直接知道前因後果。

## 轉接後的 AI 行為

轉接之後，AI 的角色變成「靜默待命」：

- 不主動發訊息給顧客
- 如果顧客繼續傳訊息給 AI，回覆：
  ```
  你的問題已經交給輔睡員處理中，他們會盡快回覆你。
  如果有其他新的問題，我也可以先幫你看看喔！
  ```
- 等到真人輔睡員在 Omnichat 後台接手後，AI 退出對話

---

## 升級路線（三級制）

```
Level 1：AI 輔睡員（自動處理 80% 的對話）
    │
    ▼ 觸發轉接
Level 2：真人輔睡員（Omnichat 後台接手）
    │
    ▼ 無法解決或嚴重客訴
Level 3：主管介入（LINE Notify 通知門市店長）
```

Level 2 → Level 3 的判斷由真人輔睡員決定，AI 不介入。
AI 只負責 Level 1 → Level 2 的轉接。

## 三家門市的轉接對應

| 情境 | 轉接對象 |
|------|---------|
| 線上訂單問題 | 總部客服群組 |
| 板橋店體驗預約 | 板橋門市群組 |
| 桃園店體驗預約 | 桃園門市群組 |
| 台中店體驗預約 | 台中門市群組 |
| 顧客未指定門市 | 依顧客地區判斷，或轉總部 |

顧客地區判斷：
- Omnichat 如有帶地址資訊 → 自動分流
- 如沒有 → AI 詢問「你方便告訴我在哪個城市嗎？」
- 如顧客不回答 → 轉總部客服
