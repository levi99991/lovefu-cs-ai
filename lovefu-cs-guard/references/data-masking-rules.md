# 個資遮蔽規則｜data-masking-rules.md

## 原則

API 回應中的個資在傳給 LLM 之前必須遮蔽。
顧客看到的 AI 回覆中也不會出現完整個資。

---

## 遮蔽對照表

| 欄位類型 | 辨識方式 | 遮蔽規則 | 輸入範例 | 輸出範例 |
|---------|---------|---------|---------|---------|
| 台灣手機 | 09 開頭 10 碼 | 前 6 碼替換為 **** | 0912345678 | ****5678 |
| 市話 | 0X-XXXXXXXX | 只保留區碼 + 後 4 碼 | 02-22520110 | 02-****0110 |
| Email | 含 @ 的字串 | 前 3 字元 + *** + @domain | che@gmail.com | che***@gmail.com |
| 地址 | 包含「路」「街」「巷」「弄」「號」 | 只保留到路/街名 | 板橋區雙十路二段110號 | 板橋區雙十路 |
| 信用卡號 | 12~19 位數字 | 完全替換 | 4111111111111111 | ************ |
| 身分證 | 1字母+9數字 | 完全替換 | A123456789 | ********** |
| 護照號碼 | 英數混合 6~9 碼 | 完全替換 | 12345678 | ******** |

---

## 不需遮蔽的欄位

以下欄位可以完整傳給 LLM 和顯示給顧客：

- 訂單編號（如 #LF2026001）
- 商品名稱（如「山丘樂眠床 5 尺」）
- 訂單金額（如 NT$14,900）
- 訂單狀態（如「已出貨」）
- 追蹤編號（物流追蹤碼）
- 會員等級（如「睡薄墊的海獺」）
- 會員點數（如「350 點」）
- 配送方式（如「宅配」）
- 下單日期

## 特殊規則

### 顧客自己提供的資訊

如果顧客在對話中主動說出自己的完整手機號碼，
AI 回覆時不需要重複完整號碼。

```
顧客：我的手機是 0912345678
AI：好的，我用 ****5678 這支手機幫你查一下。
```

### 查詢別人的資料

AI 不能查詢或顯示非對話者本人的資料。

```
顧客：幫我查我老婆的訂單
AI：為了保護個資，我只能查詢你自己帳號的訂單喔。
    如果是你太太的訂單，可以請她直接跟我們聯繫。
```

### 訂單中的收件人資訊

訂單 API 回應中可能包含收件人姓名和地址。
- 收件人姓名：可以顯示（通常就是顧客本人）
- 收件人地址：遮蔽到路/街名
- 收件人電話：遮蔽後 4 碼

---

## 在程式碼中的實作位置

遮蔽邏輯在 `scripts/api_guard.py` 的 `mask_pii()` 函式中。
此函式在每次 API 回應返回後、傳給 LLM 之前自動執行。
